## rev1 changes (`tasks/conductor/prompts.py`, `D16_REVISION = "rev1"`)

1. **Worked example embedded in each worker system prompt**, built at
   import time from the machine-verified `DEMONSTRATIONS` objects (same
   payload-text rendering the runtime uses), so the example text cannot
   drift from what the runtime accepts — the 0A
   executes-through-runtime test now transitively checks the prompt
   examples.
2. **Envelope contract stated twice** (task framing + closing line):
   "Your reply must contain `<artifact>` and `</artifact>` exactly once,
   with the expression between them."
3. **Literal-identifier rule made explicit** for Lookup and Code:
   "always write the literal word `resource` — never the resource's name
   (such as R-7K2)".
4. **Tool-computes-not-you rule for Math**: "An exact calculator
   evaluates the one expression you emit — do NOT compute the value
   yourself and do NOT show numeric working. Plain ASCII only — no
   LaTeX." Brevity instruction ("at most one short sentence of
   reasoning") on all three worker prompts, targeting the cap
   truncations.
5. `SYSTEM_DIRECT` unchanged: the direct arms run on the policy model,
   which the worker pool does not load; revising it without execution
   evidence would be a change without measurement (revisit at 1A).
6. Provenance: `D16_REVISION` added and wired into the runtime profile
   (`profile["prompts"]["d16_revision"]`, worker-visible — the 0A
   close-out names the D16 prompt revision as an execution-provenance
   field). `chat_template_bytes.json` regenerated (69 hashes; system
   prompts enter the rendered bytes, so every worker-request hash moved).

## Read-out

- **Lookup: fixed.** 0/6 → 15/15 success (envelope + literal `resource` +
  correct key/field). The worked example + duplicated envelope contract
  was sufficient for the Instruct model.
- **Math: unchanged headline, changed shape.** Still 0/15 legal
  artifacts, but truncation fell 4/4 → 8/15 and the model still writes
  step-by-step LaTeX CoT and never reaches an artifact even when it
  stops before the cap (203–241 generated tokens, `finish_reason=eos`,
  no `<artifact>` substring). The Math-Instruct model's CoT alignment
  overrides the format instruction stated once at the top and once at
  the bottom of the system prompt.
- **Code: envelope fixed, identifier not.** `E_NO_ARTIFACT` ×4 →
  `E_PARSE` ×10 with the envelope now always present — but the model
  still substitutes the salient payload-header handle for the literal
  `resource` (`count_gt(stable_unique(R-3B8), 8)`), despite the
  prohibition and the positive worked example.
